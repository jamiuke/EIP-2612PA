# eip2612pa

An **EIP-2612 Permit Auditor** for the [Pharos Network](https://pharos.xyz). Give it a contract address on Pharos Atlantic Testnet or Pacific Ocean Mainnet, and it audits the deployed bytecode for EIP-2612 correctness ‚Äî checks the `permit()` selector, verifies the `nonces()` getter, looks for the `Permit(...)` type-hash literal, and confirms the ecrecover precompile is used for signature recovery.

Ships as a [Pharos Agent Center](https://www.pharos.xyz/agent-center) skill ‚Äî drop it into Claude / Codex / OpenClaw and the agent can audit any Pharos contract's EIP-2612 implementation on demand.

## What it checks

| # | Check | What it looks for in bytecode | Severity |
|---|---|---|---:|
| 1 | `permit()` selector present | the bytecode contains `0xd505accf` | 100 |
| 2 | `nonces()` getter present | the bytecode contains `0x7ecebe00` | 100 |
| 3 | `DOMAIN_SEPARATOR()` getter present | the bytecode contains `0x3644e515` | 100 |
| 4 | `Permit(...)` type-hash literal present | the 56-byte string `Permit(address owner,address spender,uint256 value,uint256 nonce,uint256 deadline)` | 95 |
| 5 | ECDSA recovery uses `ecrecover` (precompile 0x01) | the bytecode contains a `STATICCALL` to address `0x0000...0001` | 100 |
| 6 | Nonce is incremented after use | `SLOAD` (read nonces[owner]) followed by `SSTORE` (write back) in `permit()` | 80 |
| 7 | Deadline is checked against `block.timestamp` | the `TIMESTAMP` opcode (0x42) is present near the deadline comparison | 90 |
| 8 | Owner is checked against `ecrecover` return | an `EQ` opcode (0x14) is present after the ecrecover | 90 |

Each check has one of three verdicts: **PASS**, **FAIL**, or **NOT_FOUND** (the bytecode doesn't implement EIP-2612 at all).

## Quick start

### Zero-dependency (bash + curl only)

```bash
# Default: Markdown report, mainnet
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet

# Machine-readable JSON
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet --format json

# Strict mode: exit 1 on any FAIL (useful in CI)
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet --strict
```

### Python (richer output)

```bash
pip install web3
python3 scripts/audit.py 0xYOUR_CONTRACT --network mainnet --format md
```

### Run the demo

```bash
bash scripts/audit_demo.sh
```

This audits a real public mainnet contract and prints a sample report.

### Verify the install

```bash
bash scripts/audit.sh --help
python3 scripts/audit.py --help
python3 tests/test_audit.py
```

## Honest scope

eip2612pa is a **starting point**, not a verdict. It catches the obvious EIP-2612 pitfalls but does NOT substitute for a full audit firm. Use it to triage contracts quickly, then hand off to a real audit for the ones that flag.
